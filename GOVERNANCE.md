## Decision Making

### Minor Changes
- Any contributor can submit
- One maintainer approval required
- No RFC needed

### Major Changes
- Requires RFC (Request for Comments)
- 2+ maintainer approvals
- BDFL tie-breaker if needed

### Breaking Changes
- Requires RFC
- Community feedback period (2 weeks minimum)
- Unanimous maintainer approval
- BDFL final approval

## Release Process

1. **Development** - Feature development on `develop` branch
2. **Testing** - Full test suite, security scan
3. **RC Release** - Release candidate for testing
4. **Stable Release** - Merge to `main`, tag release
5. **Announcement** - GitHub release, Discord, Twitter
